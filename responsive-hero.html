Case Study

Responsive Hero Animation with Performance Budget

A React hero component that delivers a smooth, animated experience on desktop while serving a static fallback for mobile and reduced-motion users — keeping LCP under 2.5s.

Role: Front-End Developer Focus: Performance, Accessibility, UX Tech: React.lazy, prefers-reduced-motion, SVG, Gatsby Image

The Problem

Leadership wanted a dynamic homepage hero that looked high-end, but heavy WebGL animation hurt page speed on mobile. We needed a way to show movement without sacrificing performance or accessibility.

What I Built

  • Inline SVG headline for crisp rendering and crawlable text.
  • Desktop-only animation component (AtomView) loaded with React.lazy and Suspense.
  • Static image fallback for mobile and motion-sensitive users via prefers-reduced-motion.
  • Optimized with gatsby-plugin-image for lightweight delivery.

Impact

  • Improved LCP and CLS scores while maintaining motion design quality.
  • Better accessibility and comfort for all users.
  • Reduced hero load weight by over 60% on mobile devices.
