Case Study

Live Blockchain Stats for Rootstock.io

A React component that streams live network metrics like hashrate and merged mining percentage — connected directly to blockchain data and managed via WordPress for non-technical editors.

Role: Front-End Developer Focus: Data Integration, Performance, SEO Tech: React, Socket.IO, WebSocket, REST, ACF (WordPress)

The Problem

Marketing wanted to show live network activity — hashrate, active accounts, merged mining percentage — on high-traffic pages. But these numbers changed constantly, and updating them manually led to stale, outdated data that hurt credibility.

What I Built

I built a React component called NumbersNew that connects live to Rootstock’s explorer API and WebSocket feeds.

  • Data is fetched from WebSocket and REST endpoints for accurate, real-time updates.
  • Stats are managed through WordPress ACF — marketing can add or reorder metrics without dev support.
  • Each metric updates smoothly using lightweight count-up animations.
  • All sockets are cleaned up on unmount to prevent memory leaks.

Impact

  • Marketing gained full control of live stats with no code changes.
  • Users saw trustworthy, current network data that refreshed automatically.
  • Helped Rootstock pages rank better for terms like “Bitcoin sidechain hashrate” and “Rootstock merged mining.”
